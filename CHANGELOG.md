# Changelog

## [1.1.0] — 2026-04-03

Compression rework: MCP-based with rule-based fallback. No API key required.
Lifecycle management for decisions, risks, and deferred work (Phases 4-8).

### Added
- Rule-based compression extractor (`compression/rule-based.ts`): DECISION/RISK/DEFERRED
  signal patterns run synchronously at every Stop hook, no API key needed
- `compress_session` and `submit_compression` MCP tools for AI upgrade pass
- `pending_compression` and `compression_source` columns on sessions table
- `GET /api/sessions/pending-compression` endpoint
- Context hook now appends a compression directive when a session is pending upgrade
- `/lore compress` command in lore.md and kg-query.md routing table
- Lifecycle state machine: decisions (`active|superseded|historical|archived`),
  deferred (`active|completed|abandoned|superseded`), risks (`active|mitigated|accepted|superseded|archived`)
- `lifecycle_status`, `last_reviewed_at`, `reviewed_by` on all record tables (Phase 10)
- `staleness_note` column for orphan/re-anchor explanation (Phase 11)
- `POST /api/records/lifecycle` endpoint with 7 action types
- CLI `review` command: per-type action menus (mitigate, accept, complete, abandon, still_valid)
- Lifecycle-aware context injection and graph tools (graph_decisions, graph_symbol lifecycle param)
- Advisor: zombie deferred, supersession conflict, unreviewed template records,
  unresolved conflict detectors (Phase 6)
- Workflow advisor: `no_lifecycle_review` pattern with 30-day cadence check (Phase 7)
- Structural orphan-to-review pipeline: orphaned symbols queue `pending_review=1` (Phase 7)
- Cross-repo lifecycle propagation: `lifecycle_status` on `cross_repo_index` (Phase 8)
- Dashboard lifecycle health panel per repo (Phase 8)
- `compress_session` and `submit_compression` added to allowed-tools in lore.md

### Changed
- Compression no longer requires `ANTHROPIC_API_KEY` — `@anthropic-ai/sdk` removed from worker
- Doctor `API KEYS` section: ANTHROPIC_API_KEY check replaced with compression mode check
- Dashboard `ai_compression` shows mode + pending count instead of API key status
- `provenance_trace` summary now generated by the calling agent (no inline API call)
- MCP tool count: 31 → 33

### Removed
- `@anthropic-ai/sdk` dependency from `packages/worker/package.json`

Feature-complete reasoning layer. Production-hardened with security audit,
comprehensive doctor checks, and team sync.

### Added
- Full reasoning layer: decisions, risks, deferred work, session compression,
  confidence scoring
- 26 MCP tools for Claude Code and Cursor
- 40+ CLI commands

  monorepo-services, cursor-rules)

- Advisor suite: knowledge gaps, CLAUDE.md optimisation, skill gaps,
  parallelism analysis, workflow pattern detection
- 4 specialist agents: blast-radius-checker, adr-drafter, session-handover,
  cross-repo-validator, code-reviewer
- Visual graph layer: decision hierarchy, symbol impact, portfolio dependency
  map (Mermaid, DOT, D3 interactive)
- Code annotation: annotated source view, provenance trace, annotation coverage
- Visual code review: codebase map, decision propagation, pre-commit diff overlay
- Comprehensive doctor with 15+ checks, --fix, --watch, --json
- Full help system: CLI grouped reference + /lore help in chat
- Monorepo service scoping on all MCP tools and DB queries
- Cursor rules importer
- Personal notes layer (never synced, never shared)
- Solo/team mode with interactive setup wizard
- CLAUDE.md interactive wizard

### Security
- Full security audit completed
- Command injection fix (execFileSync replacing execSync)
- Path traversal protection on all cwd-accepting endpoints
- Cursor lockfile path sanitisation
- config.json written with 0o600 permissions
- Compression deduplication guard
- Observation cap (500 per session, ring buffer eviction)
- Template directory scan cap (100 entries)
- Personal DB boundary verified — never synced under any condition

  blast radius queries use portfolio registry only
